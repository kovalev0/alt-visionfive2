## Что такое ISA и зачем это знать

ISA (Instruction Set Architecture) — контракт между программным обеспечением
и аппаратным обеспечением. Он определяет:
- какие инструкции существуют и что они делают
- сколько регистров есть и какого они размера
- как передаются аргументы функций и возвращаются результаты
- как программа обращается к операционной системе

Компилятор (`gcc`) знает ISA и генерирует машинный код по этим правилам.
Ядро Linux знает ISA и реализует системные вызовы по этим правилам.
Процессор знает ISA и исполняет инструкции по этим правилам.

Без понимания ISA невозможно читать дизассемблер, отлаживать ftrace-вывод
или понимать что происходит при переключении контекста.

## Принципы RISC-V

RISC-V — открытая модульная ISA. «Открытая» означает что спецификация
свободно доступна и любой может реализовать процессор без лицензионных
отчислений. «Модульная» — ключевое отличие от x86 и AArch64.

```
  x86_64:  монолитная ISA, ~1500 инструкций, сложилась исторически,
           много legacy (8086 → 286 → 386 → ... → x86_64).
           Невозможно убрать ни одну инструкцию — сломается совместимость.

  AArch64: чище x86, но фиксированный набор от ARM Ltd.,
           требует лицензирования.

  RISC-V:  базовый набор (I) = ~47 инструкций.
           Расширения подключаются по буквам: M, A, F, D, C, ...
           Каждое расширение — отдельная спецификация.
           Можно реализовать только нужное подмножество.
```

## Расширения RV64GC

На VisionFive2 стоят ядра U74 с ISA `RV64GC`. Расшифровка:

| Часть | Значение |
|-------|----------|
| `RV`  | RISC-V |
| `64`  | 64-битная архитектура (ширина регистров XLEN=64) |
| `G`   | General — псевдоним для `IMAFD` |
| `C`   | Compressed — 16-битные инструкции |

Расшифровка `G = IMAFD`:

| Буква | Расширение | Что добавляет |
|-------|-----------|---------------|
| `I` | Integer Base | Базовые целочисленные инструкции, загрузка/запись, ветвления |
| `M` | Multiply | Умножение и деление целых (`mul`, `div`, `rem`) |
| `A` | Atomic | Атомарные операции (`lr`, `sc`, `amo*`) — нужны для SMP |
| `F` | Float | 32-битная арифметика с плавающей точкой, регистры `f0–f31` |
| `D` | Double | 64-битная арифметика с плавающей точкой |

Расширение `C` (Compressed) — инструкции шириной 16 бит вместо стандартных
32 бит. Уменьшает размер кода примерно на 25%. GCC использует их
автоматически при компиляции для `riscv64gc`.

```bash
# ISA текущего ядра процессора
$ cat /proc/cpuinfo | grep isa
# ожидаемо: isa     : rv64imafdc_zicntr_zicsr_zifencei_zihpm_zca_zcd_zba_zbb
```

Поле `isa` показывает что поддерживает данное ядро.

Комбинация букв `imafdc` часто записывается одной буквой **G** (General-purpose).
Это стандартный набор для запуска полноценных ОС вроде Linux.

`rv64`: 64-битная архитектура.

`i`: Base Integer ISA — базовые целочисленные инструкции (сложение, сдвиги, переходы).

`m`: Integer Multiplication and Division — аппаратное умножение и деление.

`a`: Atomic Instructions — инструкции для работы с памятью в многоядерных
системах (необходимы для мьютексов и синхронизации).

`f / d`: Single/Double-Precision Floating-Point — аппаратная работа с числами
с плавающей точкой (32 и 64 бита).

`c`: Compressed Instructions — 16-битные версии популярных инструкций для
экономии памяти (уменьшают код на ~30%).

Расширения через `_z` — субрасширения:

`zicsr`: Инструкции для работы с регистрами управления и статуса (CSR).
Без этого нельзя даже считать время или управлять прерываниями.

`zifencei`: Инструкция FENCE.I для синхронизации кэша инструкций (важно
для самомодифицирующегося кода и JIT).

`zicntr / zihpm`: Поддержка счетчиков времени и аппаратных счетчиков
производительности (Perf).

`zba`: Инструкции для вычисления адресов (сдвиг + сложение).

`zbb`: Базовый набор для манипуляции битами (подсчет нулей, реверс байт и т.д.).
Сильно ускоряет работу ядра ОС.

`zca / zcd`: Уточнения для «сжатых» (compressed) инструкций, описывающие,
какие именно группы команд (целочисленные или плавающие) разрешено сжимать.

## Сравнение с x86_64 и AArch64

### Ширина инструкции

```
  x86_64:   переменная ширина, 1–15 байт.
            Декодер — один из самых сложных блоков процессора.

  AArch64:  фиксированная, 32 бита (кроме Thumb в 32-битном режиме).

  RISC-V:   32 бита (базовые) или 16 бит (C-расширение).
            Определяется по двум младшим битам: 11 = 32-бит, иначе 16-бит.
            Выравнивание: 32-битные — на 4 байта, 16-битные — на 2 байта.
```

### Количество регистров общего назначения

```
  x86_64:   16 регистров (rax, rbx, ... r15)
  AArch64:  31 регистр (x0–x30) + sp + xzr
  RISC-V:   32 регистра (x0–x31)
```

### Регистр нуля

В x86 нет отдельного нулевого регистра — для записи нуля нужна инструкция
`xor eax, eax`. В RISC-V регистр `x0` жёстко равен нулю: чтение всегда
возвращает 0, запись игнорируется. Это упрощает кодирование многих операций.

```asm
# RISC-V: псевдоинструкция mv разворачивается в:
addi a0, a1, 0    # a0 = a1 + 0

# Очистка регистра — псевдоинструкция mv a0, zero разворачивается в:
addi a0, x0, 0   # a0 = 0 + 0
```

### Обращение к памяти

```
  x86_64:   большинство инструкций могут обращаться к памяти напрямую.
            add eax, [rbx+rcx*4+8]  — одна инструкция.

  AArch64:  load/store architecture, есть пре/пост-инкремент адреса.

  RISC-V:   строгая load/store архитектура.
            Арифметика только над регистрами.
            Обращение к памяти — только явные lw/sw/ld/sd и т.д.
            Адресация: base + 12-bit signed immediate.
```

Пример — загрузить слово из памяти и прибавить 1:

```asm
# RISC-V (3 инструкции):
lw   t0, 0(a0)      # t0 = *(int32_t*)a0
addi t0, t0, 1      # t0 = t0 + 1
sw   t0, 0(a0)      # *(int32_t*)a0 = t0

# x86_64 (1 инструкция):
addl $1, (%rax)
```

RISC-V многословнее, но каждая инструкция простая и декодируется за
фиксированное время — отсюда предсказуемость латентности.

### Отсутствие флагов состояния

x86 и AArch64 имеют регистр флагов (EFLAGS / NZCV): после арифметической
операции автоматически устанавливаются биты overflow, zero, carry, negative.
Условные переходы проверяют эти флаги.

В RISC-V флагов **нет**. Условные переходы сравнивают два регистра напрямую:

```asm
# x86_64:
cmp  rax, rbx
je   label         # перейти если rax == rbx

# RISC-V:
beq  a0, a1, label # перейти если a0 == a1
blt  a0, a1, label # перейти если a0 < a1 (signed)
bltu a0, a1, label # перейти если a0 < a1 (unsigned)
```

Каждая ветка кодируется явно. Нет неявных побочных эффектов от арифметики
на поток управления.

## Форматы инструкций

Все 32-битные инструкции RISC-V имеют один из 6 форматов. Биты 6:0 —
всегда опкод.

```
  R-type:  [funct7][rs2][rs1][funct3][rd][opcode]
           Регистр ← Регистр op Регистр
           Пример: add rd, rs1, rs2

  I-type:  [imm[11:0]][rs1][funct3][rd][opcode]
           Регистр ← Регистр op Immediate(12 бит)
           Пример: addi rd, rs1, imm  ;  lw rd, imm(rs1)

  S-type:  [imm[11:5]][rs2][rs1][funct3][imm[4:0]][opcode]
           Store: Память[rs1+imm] ← rs2
           Пример: sw rs2, imm(rs1)

  B-type:  [imm[12|10:5]][rs2][rs1][funct3][imm[4:1|11]][opcode]
           Branch: if (rs1 op rs2) PC ← PC + imm
           Пример: beq rs1, rs2, offset

  U-type:  [imm[31:12]][rd][opcode]
           Загрузка 20-битной константы в старшие биты.
           Пример: lui rd, imm

  J-type:  [imm[20|10:1|11|19:12]][rd][opcode]
           Jump: rd ← PC+4; PC ← PC + imm
           Пример: jal rd, offset
```

Для курса достаточно запомнить: все immediate-значения — sign-extended,
максимальная непосредственная константа в I/S/B формате — 12 бит (±2048).
Загрузка произвольного 32/64-битного адреса выполняется комбинацией
`lui + addi` или специальной последовательностью для 64-битных констант.
