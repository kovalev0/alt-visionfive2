## Регистры общего назначения

RV64 имеет 32 целочисленных регистра шириной 64 бита. У каждого два имени:
архитектурное (`x0`–`x31`) и ABI-псевдоним, который используется в
ассемблере и компиляторе.

```
  Рег.   ABI-имя   Назначение                         Сохраняет callee?
  ──────────────────────────────────────────────────────────────────────
  x0     zero      Всегда 0, запись игнорируется       —
  x1     ra        Return Address (адрес возврата)      Нет
  x2     sp        Stack Pointer                        Да
  x3     gp        Global Pointer                       —
  x4     tp        Thread Pointer                       —
  x5     t0        Temporary / Alternate link reg       Нет
  x6     t1        Temporary                            Нет
  x7     t2        Temporary                            Нет
  x8     s0 / fp   Saved / Frame Pointer                Да
  x9     s1        Saved                                Да
  x10    a0        Аргумент 1 / Возвращаемое значение   Нет
  x11    a1        Аргумент 2 / Возвращаемое значение   Нет
  x12    a2        Аргумент 3                           Нет
  x13    a3        Аргумент 4                           Нет
  x14    a4        Аргумент 5                           Нет
  x15    a5        Аргумент 6                           Нет
  x16    a6        Аргумент 7                           Нет
  x17    a7        Аргумент 8 / Номер системного вызова Нет
  x18    s2        Saved                                Да
  x19    s3        Saved                                Да
  x20    s4        Saved                                Да
  x21    s5        Saved                                Да
  x22    s6        Saved                                Да
  x23    s7        Saved                                Да
  x24    s8        Saved                                Да
  x25    s9        Saved                                Да
  x26    s10       Saved                                Да
  x27    s11       Saved                                Да
  x28    t3        Temporary                            Нет
  x29    t4        Temporary                            Нет
  x30    t5        Temporary                            Нет
  x31    t6        Temporary                            Нет
```

Помимо этого:

- **`pc`** — Program Counter, не входит в `x0–x31`, недоступен напрямую как регистр
- **`f0–f31`** — 32 регистра с плавающей точкой (F/D расширения), 64-битные

## Соглашение о вызовах (Calling Convention)

Соглашение о вызовах — договорённость между вызывающим (`caller`) и
вызываемым (`callee`) кодом о том, как передаются аргументы, кто сохраняет
регистры и как возвращается результат.

### Передача аргументов

Первые 8 целочисленных аргументов передаются через `a0–a7`. Если аргументов
больше — они кладутся на стек.

```c
// Функция с тремя аргументами
long foo(long x, long y, long z);
// при вызове: x → a0, y → a1, z → a2
```

### Возвращаемое значение

Возвращается через `a0` (и `a1` для 128-битных значений или пар).

### Caller-saved vs Callee-saved

Это разделение определяет кто отвечает за сохранение регистров.

```
  Caller-saved (временные):  t0–t6, a0–a7, ra
  ─────────────────────────────────────────────────────────────
  Если в регистре t0 лежит нужное значение и вызывается какая-то
  функция — вызывающий код обязан сам сохранить t0 до вызова
  (push на стек). Вызываемая функция может затирать t0–t6 и
  a0–a7 как угодно.

  Callee-saved (сохраняемые): s0–s11, sp
  ─────────────────────────────────────────────────────────────
  Функция, использующая s0–s11, обязана сохранить их в прологе
  и восстановить в эпилоге. После возврата из функции эти
  регистры содержат те же значения что были до вызова.
```

### Стек

Стек растёт **вниз** (от старших адресов к младшим). `sp` всегда указывает
на последний занятый элемент стека (top). При входе в функцию `sp`
уменьшается на размер фрейма, при выходе — увеличивается обратно.

```
  Высокие адреса
  ┌──────────────┐
  │  аргументы   │  (если > 8, кладёт caller)
  ├──────────────┤  ← sp до вызова (= fp текущего фрейма)
  │  saved ra    │  (адрес возврата, сохраняет callee)
  ├──────────────┤
  │  saved s0    │  (если использует callee)
  ├──────────────┤
  │  saved s1    │
  ├──────────────┤
  │  локальные   │
  │  переменные  │
  ├──────────────┤  ← sp (текущий top стека)
  │  ...         │
  Низкие адреса
```

## Пролог и эпилог функции

Стандартный пролог/эпилог функции, которая использует `s0` и делает вызовы:

```asm
# Пролог: сохранить ra и s0, сдвинуть sp
my_func:
    addi sp, sp, -16     # выделить 16 байт на стеке
    sd   ra, 8(sp)       # сохранить адрес возврата
    sd   s0, 0(sp)       # сохранить s0

    # ... тело функции ...

# Эпилог: восстановить, вернуться
    ld   ra, 8(sp)       # восстановить ra
    ld   s0, 0(sp)       # восстановить s0
    addi sp, sp, 16      # освободить стек
    ret                  # = jalr x0, ra, 0
```

`ret` — псевдоинструкция для `jalr x0, 0(ra)`: переход по адресу в `ra`,
результат перехода (PC+4) записывается в `x0` (игнорируется).

## CSR: Control and Status Registers

Помимо `x0–x31`, процессор имеет набор специальных регистров — CSR
(Control and Status Registers). Они недоступны через обычные инструкции
(`add`, `lw` и т.д.), только через специальные:

```asm
csrr  t0, cycle      # прочитать счётчик тактов → t0
csrr  t0, instret    # прочитать счётчик выполненных инструкций → t0
csrr  t0, time       # прочитать системное время
csrw  mscratch, t0   # записать t0 → CSR mscratch (M-mode only)
```

Наиболее важные CSR для курса:

| CSR | Режим | Назначение |
|-----|-------|------------|
| `mstatus` | M | Глобальный статус процессора (включая IE — interrupt enable) |
| `mtvec` | M | Адрес обработчика прерываний/исключений (M-mode) |
| `mepc` | M | Адрес прерванной инструкции (M-mode exception PC) |
| `mcause` | M | Причина исключения или прерывания |
| `sstatus` | S | То же для S-mode |
| `stvec` | S | Адрес обработчика для S-mode (Linux) |
| `sepc` | S | Прерванный PC для S-mode |
| `scause` | S | Причина исключения в S-mode |
| `satp` | S | Указатель на корень таблицы страниц (MMU) |
| `cycle` | U | Счётчик тактов (read-only для userspace) |
| `time` | U | Системное время |

Linux-ядро работает в S-mode и использует `stvec`, `sepc`, `scause`.
Это будет разбираться детальнее в следующих планируемых уроках.

## Практика: регистры живого процесса

```bash
# Запустить простую программу в фоне
$ cat /dev/zero > /dev/null &
PID=$!

# Посмотреть регистры через GDB
$ gdb -p $PID -batch -ex "info registers" 2>/dev/null | head -40

$ kill $PID
```

Или через GDB после компиляции программы из следующего модуля:

```bash
$ gdb ./hello_riscv
(gdb) starti           # остановиться на первой инструкции
(gdb) info registers   # все регистры
(gdb) disas            # дизассемблировать текущую функцию
```
